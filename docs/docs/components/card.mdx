# Card

Card is a content container with optional cover image, body, and footer. It is similar to Panel but optimized for browsable content items — articles, jobs, profiles, courses.

## HTML structure

```html
<div class="card card--hover">
  <div class="card__cover"></div>
  <div class="card__body">
    <div class="card__title">Article Title</div>
    <p class="card__desc">Supporting description text.</p>
    <div class="card__meta">
      <span>Author</span>
      <span>5 min read</span>
    </div>
  </div>
  <div class="card__footer">
    <span>142 reactions</span>
    <span>32 comments</span>
  </div>
</div>
```

<DocsCallout type="info">
  Card intentionally omits `.panel`'s `overflow: hidden` on the body. If you need a card that clips child elements (e.g. an avatar overflowing a cover), add `overflow: hidden` to `.card__cover` instead.
</DocsCallout>
