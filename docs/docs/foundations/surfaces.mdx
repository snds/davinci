# Surfaces

Surfaces are the modern mapping of DaVinci's original **Containers** — semantic
regions defined by their *role in a containment hierarchy* rather than by a fixed
appearance. They exist to compose layouts without resorting to deeply nested,
arbitrarily styled boxes.

The hierarchy is:

> **Canvas › Container › ( Module · Bristol ) › content**

The shadcn/Tailwind bridge exposes the inverse surface as `bg-surface-inverse` /
`text-surface-inverse-foreground`. The other three map to existing shadcn names:
`bg-background` (Canvas), `bg-card` (Container), `bg-secondary` (Module).

## DaVinci heritage

| DaVinci name | Role | Modern token | `<Surface variant>` |
|---|---|---|---|
| **Canvas** | Master page background | `--bg` + `--bg-pattern` | `canvas` |
| **Container** | Content region on a canvas | `--bg-surface` | `container` |
| **Module** | Quiet grouping of content blocks | `--bg-subtle` | `module` |
| **Bristol** | Inverse/contrast grouping | `--bg-inverse` | `bristol` |
| ~~**Punchcut**~~ | *(retired)* Textured inset/outset region | — | — |

**Bristol & theme polarity.** In the original, Module and Bristol were used
together on a light page for within-theme contrast — Module quiet, Bristol a
dark region that pops. In the modern system Bristol is an *inverse surface*
(cf. Material's `inverseSurface`): it renders dark on a light theme and light on
a dark theme, so the "pop against the page" intent survives in both modes.

**Why Punchcut was dropped.** Punchcut added raster texture and an inset/recessed
treatment. Flat surfaces plus the elevation (`--shadow-*`) ramp cover its use
cases without shipping textures, so it was retired during modernization.

## The Surface primitive

`<Surface>` applies the right tokens and encodes the containment hierarchy. The
nesting rules are enforced with **dev-only console warnings** — never fatal — so
a Module rendered without a Container ancestor, or a nested Canvas, is flagged in
development but never breaks production.

```jsx
import { Surface } from '@davinci/ui/components/ui/surface';

<Surface variant="canvas">
  <Surface variant="container">
    <Surface variant="module">…quiet grouping…</Surface>
    <Surface variant="bristol">…contrast grouping…</Surface>
  </Surface>
</Surface>
```

## Usage guidelines

- **One Canvas per page** — it's the root; don't nest it.
- **Containers sit on the Canvas** — not inside Modules or Bristols.
- **Module vs Bristol** — reach for Module to group quietly; reach for Bristol when
  a region should stand apart from its surroundings.
- **Prefer surfaces over ad-hoc boxes** — the ladder exists specifically to avoid
  deeply nested, one-off styled containers.
