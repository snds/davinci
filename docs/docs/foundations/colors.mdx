# Colors

Davinci's color system is built on [Radix Colors](https://www.radix-ui.com/colors) — a mathematically balanced, perceptually uniform palette. Each scale has 12 steps with predictable meaning across both light and dark themes.

## Scale anatomy

Every scale follows this structure:

| Steps | Purpose |
|---|---|
| 1–2 | Page and subtle backgrounds |
| 3–5 | Component backgrounds (default, hover, active) |
| 6–8 | Borders (subtle, default, strong) |
| 9–10 | Solid fills — badges, buttons, interactive elements |
| 11 | Low-contrast text on colored backgrounds |
| 12 | High-contrast text |

## Palette scales

### Sand — neutral

Sand is the neutral backbone of the entire UI. Used for backgrounds, surfaces, borders, and body text. All `bg-*`, `fg-*`, and `border-*` semantic tokens resolve to sand values.

## Usage guidelines

**Do** use semantic tokens in all component code:
```css
/* Good */
.card { background: var(--bg-surface); border: 1px solid var(--border); }

/* Avoid — breaks light/dark theming */
.card { background: #222221; border: 1px solid #3b3a37; }
```

**Do** use raw scale values only in the token definitions themselves (in `colors_and_type.css`).

**Do** use `--accent-subtle` and `--accent-border` together for tinted container regions (e.g. info banners, highlighted rows).

**Avoid** mixing sand and blue raw values as background/foreground pairs outside of semantic token overrides.
