# Colors

Davinci's color system has two distinct layers:

1. **Foundation palette** — all 31 [Radix Colors](https://www.radix-ui.com/colors) scales, each with 12 mathematically balanced steps. These are the raw color values. Never use them directly in component code.
2. **Semantic tokens** — purpose-driven aliases over the palette that adapt automatically between dark and light themes. Always use these in components.

---

## Scale anatomy

Every Radix scale follows this step structure:

| Steps | Purpose | Example use |
|-------|---------|-------------|
| 1–2   | Page and subtle backgrounds | `--bg`, `--bg-subtle` |
| 3–5   | Component fills (default, hover, active) | `--bg-surface`, `--bg-hover` |
| 6–8   | Borders — subtle, default, strong | `--border`, `--border-strong` |
| 9–10  | Solid fills — buttons, badges, solid icons | `--accent`, `--alt` |
| 11    | Low-contrast foreground in tinted contexts | `--accent-fg`, `--alt-fg` |
| 12    | High-contrast text | `--fg` |

> **Scales invert between themes.** In dark mode, step 1 is nearly black and step 12 is near-white. In light mode the reverse is true. Semantic tokens handle this automatically — never hard-code a step number in component CSS.

## Brand colors

Davinci uses two brand colors drawn from the palette. Each has a dedicated set of semantic aliases.

### Primary — Blue

The primary interactive color. All buttons, links, focus rings, and selected states draw from the blue scale via `--accent-*` tokens.

<ColorSwatch scale="blue" title="Blue scale (--blue-1 through --blue-12)" />

| Token | Step | Role |
|-------|------|------|
| `--accent` | blue-9 | Primary fill — buttons, active badges |
| `--accent-hover` | blue-10 | Hover state |
| `--accent-subtle` | blue-3 | Tinted region background |
| `--accent-fg` | blue-11 | Text in tinted contexts |
| `--accent-border` | blue-7 | Border in tinted containers |
| `--bg-selected` | blue-4 | Selected item background |
| `--border-focus` | blue-8 | Keyboard focus ring |

### Alt — Yellow (Spotlight)

The secondary brand color. Used once per page maximum — CTA buttons, the Davinci logo dot, editorial highlights.

## Usage rules

**Always use semantic tokens in component CSS:**
```css
/* ✓ Correct — theme-aware */
.card {
  background: var(--bg-surface);
  border: 1px solid var(--border);
  color: var(--fg);
}

/* ✗ Avoid — breaks theming */
.card {
  background: #222221;
  border: 1px solid #3b3a37;
}
```

**Raw scale values belong only in token definitions** (`colors_and_type.css`). Never reference `--blue-9` directly in a component — use `--accent` instead.

**Alt (yellow) is one-per-page.** It's a spotlight signal — diluted by overuse. Reserve it for the single most important CTA on each screen.

**For tinted regions**, always pair `--*-subtle` (background) with `--*-border` (border) and `--*-fg` (text) from the same family:
```css
.info-banner {
  background: var(--accent-subtle);
  border: 1px solid var(--accent-border);
  color: var(--accent-fg);
}
```
