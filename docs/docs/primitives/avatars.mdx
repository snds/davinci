# Avatars

Avatars represent a person or entity throughout the UI. They support initials with gradient backgrounds and image photos. All avatars are circular (`--radius-full`).

<LiveDemo

## Sizes

| Size class | Diameter | Font size | Use case |
|---|---|---|---|
| `.avatar--32` | 32px | 12px | Nav, sidebars, card bylines |
| `.avatar--40` | 40px | 13px | Default — post author, feed cards |
| `.avatar--48` | 48px | 15px | Emphasized cards, hovercard |
| `.avatar--64` | 64px | 20px | Profile header |
| `.avatar--128` | 128px | 42px | Profile page hero |

## Gradient variants

When a user has no photo, initials are displayed on one of six gradient backgrounds. Assign them deterministically (e.g. `userId % 6`) so each user always gets the same color.

|---|---|
| `.avatar` (default) | Blue 7 → Blue 9 |
| `.avatar--g2` | Yellow 7 → Yellow 9 |

| `.avatar--g4` | Mauve 7 → Mauve 9 |
| `.avatar--g5` | Teal 7 → Teal 9 |
| `.avatar--g6` | Bronze 7 → Bronze 9 |

## Photo avatar

Set `background` to transparent and add an `<img>` child. The `overflow: hidden` on `.avatar` clips the image to a circle.

```html
<div class="avatar avatar--48">
  <img src="/path/to/photo.jpg" alt="Jane Doe"
       style="width:100%;height:100%;object-fit:cover;" />
</div>
```

<DocsCallout type="tip" title="Accessibility">
Always add `alt` text to photo avatars. For initial-based avatars, the initials are decorative — wrap the avatar in an element with `aria-label="User name"` when the name isn't adjacent in the DOM.
</DocsCallout>
