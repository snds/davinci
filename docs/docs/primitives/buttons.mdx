# Buttons

Buttons are the primary interactive trigger. Davinci provides four semantic variants and three sizes. Every button shares the same base shape: `--radius-md` corners, Inter Medium label, and a `--shadow-focus` focus ring.

<LiveDemo
  title="All button variants"
  html={`<div style="display:flex;gap:10px;flex-wrap:wrap;align-items:center;">
  <button class="btn btn--primary">Primary</button>
  <button class="btn btn--secondary">Secondary</button>
  <button class="btn btn--outline">Outline</button>
  <button class="btn btn--ghost">Ghost</button>
</div>`}
  pad="28px 24px"
/>

## Variants

| Variant | Token usage | When to use |
|---|---|---|
| **Primary** | `--accent` fill, `--fg-on-accent` text | The single most important action on a view |
| **Secondary** | `--bg-surface` fill, `--border` stroke, `--fg` text | Complementary actions, secondary CTAs |
| **Outline** | Transparent fill, `--accent-border` stroke, `--accent-fg` text | Moderate-emphasis action in an accent context |
| **Ghost** | No fill, no border, `--fg-muted` text | Low-emphasis — cancel, back, tertiary nav |

### Primary

Solid `--accent` (`--blue-9`) fill. White text. On hover, deepens to `--accent-hover` (`--blue-10`). Use for the most important action in a region — one primary button per section maximum.

<LiveDemo
  title=".btn.btn--primary"
  html={`<div style="display:flex;gap:8px;align-items:center;">
  <button class="btn btn--primary">Publish Post</button>
  <button class="btn btn--primary" disabled>Disabled</button>
</div>`}
  pad="20px 24px"
/>

```html
<button class="btn btn--primary">Publish Post</button>
```

### Secondary

Surface-colored fill with a 1px `--border` stroke. Used alongside Primary when a second action is needed at equal prominence (e.g. "Save Draft" next to "Publish").

<LiveDemo
  title=".btn.btn--secondary"
  html={`<div style="display:flex;gap:8px;align-items:center;">
  <button class="btn btn--secondary">Save Draft</button>
  <button class="btn btn--secondary" disabled>Disabled</button>
</div>`}
  pad="20px 24px"
/>

```html
<button class="btn btn--secondary">Save Draft</button>
```

### Outline

Transparent fill, `--accent-border` stroke, `--accent-fg` text. Use in blue-tinted contexts (info banners, selected state regions) where the solid Primary would feel too heavy.

<LiveDemo
  title=".btn.btn--outline"
  html={`<div style="display:flex;gap:8px;align-items:center;">
  <button class="btn btn--outline">View Details</button>
  <button class="btn btn--outline" disabled>Disabled</button>
</div>`}
  pad="20px 24px"
/>

```html
<button class="btn btn--outline">View Details</button>
```

### Ghost

No fill, no border. Relies on text color alone. Used for cancel actions, tertiary controls, navigation items that behave like buttons. On hover, shows a subtle `--bg-hover` tint.

  <button class="btn btn--ghost" disabled>Disabled</button>
</div>`}
  pad="20px 24px"

```html
<button class="btn btn--ghost">Cancel</button>
```

## Icon buttons

Add a Material Symbol icon before the label using `<span class="material-symbols-rounded">`. The icon inherits button size. For a circular icon-only button, add `.btn--icon`.

<LiveDemo
  title="Icon buttons"
  html={`<div style="display:flex;gap:10px;align-items:center;flex-wrap:wrap;">
  <button class="btn btn--primary">
    <span class="material-symbols-rounded" style="font-size:16px;">add</span>
    New Post
  </button>
  <button class="btn btn--secondary">
    <span class="material-symbols-rounded" style="font-size:16px;">bookmark</span>
    Save
  </button>
  <button class="btn btn--ghost btn--icon" aria-label="Edit">
    <span class="material-symbols-rounded">edit</span>
  </button>
  <button class="btn btn--ghost btn--icon" aria-label="More">
    <span class="material-symbols-rounded">more_horiz</span>
  </button>
</div>`}
  pad="20px 24px"
/>

```html
<button class="btn btn--primary">
  <span class="material-symbols-rounded" style="font-size:16px;">add</span>
  New Post
</button>

<button class="btn btn--ghost btn--icon" aria-label="Edit">
  <span class="material-symbols-rounded">edit</span>
</button>
```

## Usage do's and don'ts

<DocsCallout type="tip" title="Do">
Use at most one Primary button per visible region. Pair Primary + Ghost for confirm/cancel dialogs. Always include visible text with icon buttons unless space is genuinely constrained — in that case add `aria-label`.
</DocsCallout>

<DocsCallout type="danger" title="Don't">
Don't use Primary for destructive actions. Don't place two Primary buttons side by side. Don't use Ghost for the primary action on a page.
</DocsCallout>
